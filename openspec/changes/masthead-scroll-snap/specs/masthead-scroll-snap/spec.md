## ADDED Requirements

### Requirement: Scroll snap to clean boundary after idle in masthead zone

When the user stops scrolling with `scrollY` between `0` (exclusive) and the masthead section's `offsetHeight` (exclusive), the block SHALL wait 300ms and then smoothly scroll to the nearest clean boundary:

- If `scrollY / mastheadHeight < 0.5`: snap to `scrollY = 0` (masthead fully visible)
- If `scrollY / mastheadHeight >= 0.5`: snap to `scrollY = mastheadHeight` (masthead fully hidden, header at top)

The snap SHALL use `window.scrollTo({ top: target, behavior: 'smooth' })`.

#### Scenario: Scroll stops below 50% coverage snaps down

- **WHEN** the user stops scrolling with `scrollY` at 30% of masthead height
- **AND** 300ms pass with no further scroll events
- **THEN** the page SHALL smooth-scroll to `scrollY = 0` (full masthead visible)

#### Scenario: Scroll stops at or above 50% coverage snaps up

- **WHEN** the user stops scrolling with `scrollY` at 60% of masthead height
- **AND** 300ms pass with no further scroll events
- **THEN** the page SHALL smooth-scroll to `scrollY = mastheadHeight` (masthead fully hidden)

#### Scenario: Scroll at exact boundary does not trigger snap

- **WHEN** `scrollY` is `0` or `scrollY >= mastheadHeight`
- **THEN** no snap SHALL be triggered regardless of idle time

#### Scenario: Content zone scroll does not trigger snap

- **WHEN** the user is scrolling through content below the masthead (`scrollY >= mastheadHeight`)
- **THEN** no snap logic SHALL execute

---

### Requirement: Snap does not interfere with active user scrolling

The snap logic SHALL NOT fire while the user is actively scrolling. Each scroll event SHALL reset the 300ms debounce timer. The snap only evaluates after 300ms of complete scroll inactivity.

#### Scenario: Continuous scrolling never triggers snap

- **WHEN** the user scrolls continuously without stopping for 300ms
- **THEN** no snap SHALL be triggered during the scroll

#### Scenario: Debounce resets on each scroll event

- **WHEN** the user scrolls, stops for 200ms, then scrolls again
- **THEN** the 300ms timer SHALL restart from the second scroll event
- **AND** no snap SHALL have fired during the 200ms pause

---

### Requirement: Programmatic snap does not re-trigger snap evaluation

While a snap animation is in progress (`isSnapping = true`), scroll events generated by the programmatic `scrollTo` SHALL NOT reset the debounce timer or trigger a new snap evaluation. The `isSnapping` flag SHALL be cleared when the `scrollend` event fires.

#### Scenario: Snap animation does not cause feedback loop

- **WHEN** a snap scrollTo is in progress
- **AND** the programmatic scroll fires scroll events
- **THEN** those scroll events SHALL be ignored by the snap debounce logic

#### Scenario: isSnapping clears on scrollend

- **WHEN** a snap animation completes (or is interrupted by user input)
- **AND** the `scrollend` event fires
- **THEN** `isSnapping` SHALL be set to `false`
- **AND** subsequent user scroll events SHALL be processed normally

---

### Requirement: User interaction cancels active snap

When the user initiates a scroll gesture (wheel, touch, keyboard) while a snap animation is in progress, the browser's native smooth scroll cancellation SHALL stop the programmatic scroll. The `scrollend` event SHALL fire, clearing `isSnapping`, and the debounce timer SHALL resume processing user scroll events.

#### Scenario: User wheel during snap cancels it

- **WHEN** a snap animation is scrolling toward `scrollY = 0`
- **AND** the user scrolls with the mouse wheel
- **THEN** the programmatic scroll SHALL stop
- **AND** after `scrollend` fires and 300ms idle, a new snap evaluation SHALL occur
